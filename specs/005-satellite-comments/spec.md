## User Scenarios & Testing *(mandatory)*

### User Story 1 - 게시글에 댓글 작성 (Priority: P1)

사용자가 게시글(행성)을 열람한 뒤, 해당 게시글에 대한 의견이나 피드백을 댓글로 남길 수 있다. 댓글을 작성하면 행성 주위를 도는 인공위성이 새로 생성되어 3D 공간에서 시각적으로 표현된다.

**Why this priority**: 댓글은 커뮤니티의 핵심 상호작용 수단이다. 별주기(좋아요)만으로는 구체적인 피드백을 전달할 수 없으므로, 텍스트 기반 소통 채널이 최우선으로 필요하다.

**Independent Test**: 게시글 상세 화면에서 댓글을 작성하고, 작성된 댓글이 목록에 표시되며 3D 공간에 인공위성이 추가되는 것을 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 게시글 상세 화면을 보고 있을 때, **When** 댓글 내용을 입력하고 제출하면, **Then** 댓글이 목록에 즉시 표시되고 행성 주위에 새 인공위성이 나타난다
2. **Given** 사용자가 댓글을 작성할 때, **When** 닉네임과 내용을 입력하지 않고 제출하면, **Then** 유효성 검증 메시지가 표시되고 댓글이 저장되지 않는다
3. **Given** 게시글에 여러 댓글이 있을 때, **When** 게시글 상세 화면을 열면, **Then** 모든 댓글이 시간순으로 표시되고 각 댓글에 대응하는 인공위성이 행성 주위를 공전한다

---

### User Story 2 - 인공위성 클릭 시 댓글 포커싱 (Priority: P1)

사용자가 3D 공간에서 행성 주위를 도는 인공위성을 클릭하면, 해당 인공위성에 연결된 댓글이 사이드 패널에서 하이라이트되어 포커싱된다. 이를 통해 3D 시각화와 댓글 목록 간의 직관적인 연결을 제공한다.

**Why this priority**: 인공위성 시각화의 핵심 가치는 3D 오브젝트와 댓글 간의 상호작용이다. 클릭-포커싱 없이는 인공위성이 단순 장식에 불과하므로 댓글 작성과 동일한 우선순위로 구현해야 한다.

**Independent Test**: 인공위성을 클릭했을 때 대응하는 댓글이 사이드 패널에서 스크롤되어 하이라이트되는 것을 확인한다.

**Acceptance Scenarios**:

1. **Given** 행성 주위에 여러 인공위성이 공전하고 있을 때, **When** 특정 인공위성을 클릭하면, **Then** 해당 댓글이 사이드 패널에서 하이라이트되고 자동 스크롤된다
2. **Given** 인공위성이 하이라이트된 상태일 때, **When** 다른 인공위성을 클릭하면, **Then** 이전 하이라이트가 해제되고 새로 클릭한 인공위성의 댓글이 포커싱된다
3. **Given** 사이드 패널에서 댓글을 클릭할 때, **When** 해당 댓글을 선택하면, **Then** 3D 공간에서 대응하는 인공위성이 시각적으로 강조된다

---

### User Story 3 - 대댓글(답글) 작성 (Priority: P2)

사용자가 기존 댓글에 대한 답글(대댓글)을 작성할 수 있다. 대댓글은 부모 댓글 아래에 들여쓰기되어 표시되며, 부모 인공위성 근처에 더 작은 인공위성으로 시각화된다.

**Why this priority**: 대댓글은 토론의 깊이를 더하지만, 기본 댓글 기능 없이는 동작하지 않으므로 P2로 설정한다. 단, 커뮤니티 소통 품질을 높이는 데 중요하다.

**Independent Test**: 기존 댓글에 답글을 작성하고, 부모 댓글 아래에 들여쓰기되어 표시되며 3D 공간에서 부모 인공위성 근처에 작은 위성으로 나타나는 것을 확인한다.

**Acceptance Scenarios**:

1. **Given** 댓글 목록에서 특정 댓글을 보고 있을 때, **When** "답글" 버튼을 누르고 내용을 입력하면, **Then** 대댓글이 부모 댓글 아래에 들여쓰기되어 표시된다
2. **Given** 대댓글이 작성되었을 때, **When** 3D 공간을 보면, **Then** 부모 인공위성 근처에 더 작은 인공위성이 나타난다
3. **Given** 대댓글의 인공위성을 클릭할 때, **When** 클릭하면, **Then** 사이드 패널에서 해당 대댓글이 포커싱된다

---

### Edge Cases

- 댓글 내용이 비어있거나 최대 길이를 초과하면 어떻게 되는가? → 유효성 검증으로 차단하고 안내 메시지 표시
- 행성에 댓글이 상한(50개)에 도달하면? → 댓글 작성 UI 비활성화 및 안내 메시지 표시
- 대댓글의 깊이가 깊어지면? → 1단계 대댓글만 허용 (댓글 → 대댓글, 대댓글에 대댓글 불가)
- 삭제된 댓글에 대댓글이 있으면? → "삭제된 댓글입니다" 표시로 대체하고 대댓글은 유지
- 비로그인 사용자가 댓글을 작성하려면? → 닉네임 입력으로 비회원 댓글 허용 (기존 별주기와 동일한 정책)

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 게시글(행성)에 대한 댓글을 작성, 조회할 수 있어야 한다
- **FR-002**: 댓글 작성 시 닉네임(1~20자)과 내용(1~500자)을 필수로 입력해야 한다
- **FR-003**: 시스템은 1단계 대댓글(답글)을 지원해야 한다 (대댓글에 대한 대댓글은 불허)
- **FR-004**: 각 댓글은 3D 공간에서 행성 주위를 도는 인공위성으로 시각화되어야 한다. 인공위성은 별(금색 구체)과 시각적으로 구분되어야 하며, 은색 계열 색상 + 비구체 형태(원통/큐브) + 혜성 스타일 꼬리 이펙트로 표현한다. 궤도는 별보다 바깥에 위치한다
- **FR-005**: 대댓글은 부모 댓글의 인공위성 근처에 더 작은 위성으로 시각화되어야 한다
- **FR-011**: 행성당 댓글 수는 최대 50개로 제한하며, 상한 도달 시 사용자에게 안내 메시지를 표시해야 한다
- **FR-006**: 인공위성을 클릭하면 사이드 패널에서 해당 댓글이 하이라이트되고 자동 스크롤되어야 한다
- **FR-007**: 사이드 패널에서 댓글을 클릭하면 3D 공간의 대응 인공위성이 시각적으로 강조되어야 한다
- **FR-008**: 댓글 목록은 작성 시간순으로 정렬되어야 한다
- **FR-009**: 비로그인 사용자도 닉네임 입력으로 댓글을 작성할 수 있어야 한다 (optional auth)
- **FR-010**: 로그인 사용자가 댓글을 작성하면 사용자 정보가 자동으로 연결되어야 한다

### Key Entities

- **Comment (댓글)**: 게시글에 대한 텍스트 피드백. 작성자 닉네임, 내용, 작성 시간, 소속 행성, 부모 댓글(대댓글인 경우) 정보를 포함
- **Satellite (인공위성)**: 댓글의 3D 시각적 표현. 행성 주위를 공전하며, 댓글과 1:1 대응. 대댓글은 부모 위성 근처에 더 작은 크기로 표현

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 댓글 작성을 10초 이내에 완료할 수 있다
- **SC-002**: 인공위성 클릭 후 0.5초 이내에 해당 댓글이 사이드 패널에서 포커싱된다
- **SC-003**: 행성당 최대 50개의 댓글(인공위성)이 있어도 3D 렌더링이 30fps 이상 유지된다
- **SC-004**: 대댓글을 포함한 댓글 스레드가 직관적으로 구분되어 사용자가 대화 흐름을 쉽게 파악할 수 있다
- **SC-005**: 3D 인공위성과 사이드 패널 댓글 간의 양방향 연결이 100% 정확하게 동작한다

## Assumptions

- 기존 별주기(Star) 시스템과 동일하게 비회원도 닉네임만으로 댓글 작성 가능 (optional auth 패턴 재사용)
- 대댓글 깊이는 1단계로 제한하여 UI 복잡도를 관리 가능한 수준으로 유지
- 인공위성 렌더링은 기존 Star3D의 InstancedMesh 패턴을 참고하여 성능 최적화
- 댓글 삭제/수정 기능은 이번 스코프에 포함하지 않음 (향후 별도 기능으로 추가)
- 댓글 신고/차단 기능은 이번 스코프에 포함하지 않음
