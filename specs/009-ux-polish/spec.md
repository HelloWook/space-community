# Feature Specification: UX 개선 — WASD 자유이동, 모달 리디자인, 우주 장식 강화

**Feature Branch**: `009-ux-polish`
**Created**: 2026-03-27
**Status**: Draft
**Input**: User description: "1. WASD 이동이 제자리로 돌아오는 문제 — 자유롭게 이동하고 싶음. 2. 댓글/작성 모달이 너무 좁고 디자인이 별로 — shadcn 기반으로 사용성과 미감 향상. 3. 우주가 너무 비어 보임 — 다양한 에셋으로 우주를 풍성하게."

## User Scenarios & Testing *(mandatory)*

### User Story 1 — WASD 자유 탐험 (Priority: P1)

사용자가 우주 뷰에서 WASD 키로 카메라를 이동하면, 카메라가 이동한 위치에 그대로 머무른다. 키를 놓아도 원래 위치로 되돌아가지 않으며, 사용자는 우주 공간을 자유롭게 탐험할 수 있다. 은하(행성)를 클릭하면 해당 위치로 카메라가 부드럽게 전환되고, 전환이 끝나면 다시 WASD로 자유 이동이 가능하다.

**Why this priority**: 현재 이동 후 카메라가 원점으로 끌려오는 현상은 핵심 인터랙션을 완전히 망가뜨린다. 우주 탐험이 이 서비스의 핵심 경험이므로 가장 먼저 해결해야 한다.

**Independent Test**: WASD로 카메라를 임의 위치로 이동시킨 뒤 5초 대기 — 카메라 위치가 변하지 않으면 성공.

**Acceptance Scenarios**:

1. **Given** 우주 뷰에 진입한 상태, **When** W키를 3초간 누르고 손을 뗌, **Then** 카메라가 전진한 위치에 정지해 있으며 원점으로 돌아가지 않는다
2. **Given** WASD로 원점에서 멀리 이동한 상태, **When** 은하를 클릭, **Then** 카메라가 해당 은하로 부드럽게 전환된다
3. **Given** 은하 전환 애니메이션 중, **When** WASD를 누름, **Then** 전환이 완료될 때까지 WASD 입력이 무시된다
4. **Given** 은하 전환이 완료된 상태, **When** WASD를 누름, **Then** 새 위치 기준으로 자유 이동이 가능하다

### User Story 2 — 모달 리디자인 (Priority: P1)

모든 오버레이(게시글 작성, 은하 생성, 게시글 상세, 댓글 작성, 로그인)가 현재 400px 고정 너비에서 벗어나, 콘텐츠 유형에 맞는 적절한 크기와 세련된 디자인을 갖춘다. shadcn Dialog 컴포넌트를 기반으로 일관된 디자인 시스템을 적용하며, 닫기 버튼(X), 진입/퇴장 애니메이션, 반응형 크기를 제공한다.

**Why this priority**: 모달은 콘텐츠 작성과 소비의 핵심 인터페이스이다. 좁은 400px 모달에서 3D 프리뷰와 커스터마이즈 패널을 사용하는 것은 매우 불편하다.

**Independent Test**: 각 모달을 열어 콘텐츠가 잘리지 않고 읽기 편한지 확인하며, 닫기 버튼·ESC·배경 클릭으로 닫히는지 테스트.

**Acceptance Scenarios**:

1. **Given** 게시글 작성 폼을 열었을 때, **When** 3D 행성 프리뷰와 커스터마이즈 패널이 표시됨, **Then** 모든 요소가 잘리지 않고 여유 있게 배치된다
2. **Given** 게시글 상세 모달을 열었을 때, **When** 긴 글과 댓글 목록이 있음, **Then** 스크롤이 자연스럽고 읽기 편한 너비를 가진다
3. **Given** 모달이 열려 있을 때, **When** 우측 상단 X 버튼을 클릭, **Then** 부드러운 퇴장 애니메이션과 함께 모달이 닫힌다
4. **Given** 모바일 화면(768px 미만)에서 모달을 열었을 때, **When** 화면에 표시됨, **Then** 화면 폭에 맞게 반응형으로 조정된다

### User Story 3 — 우주 장식 강화 (Priority: P2)

우주 공간이 현재보다 훨씬 풍성하고 생동감 있게 느껴진다. 성운(네뷸라), 우주 먼지, 추가 소행성대, 원거리 은하 배경 등 다양한 시각 요소가 추가되어 사용자가 실제 우주를 탐험하는 듯한 몰입감을 느낀다.

**Why this priority**: 공간의 풍성함은 서비스의 세계관과 몰입감에 크게 기여하지만, 기능적 문제(P1)가 먼저 해결되어야 탐험할 공간이 의미를 갖는다.

**Independent Test**: 우주 뷰에 진입하여 360도 둘러보았을 때, 어느 방향에서도 빈 공간이 지배적이지 않으면 성공.

**Acceptance Scenarios**:

1. **Given** 우주 뷰에 진입한 상태, **When** 카메라를 360도 회전, **Then** 모든 방향에서 성운·별·소행성 등 시각 요소가 보인다
2. **Given** 우주 뷰를 보고 있을 때, **When** 시간이 경과, **Then** 유성이 지나가고 성운이 은은하게 빛나며 살아있는 느낌을 준다
3. **Given** 장식 요소가 추가된 상태, **When** 카메라를 빠르게 이동, **Then** 프레임 드롭 없이 부드럽게 렌더링된다 (60fps 유지)

---

### User Story 4 — 모달 접근성 (Priority: P3)

모달이 열렸을 때 키보드 포커스가 모달 내부로 이동하고, Tab 키로 모달 내 요소를 순환할 수 있으며, 모달 뒤의 요소는 포커스 불가능하다. 스크린 리더가 모달 역할과 제목을 올바르게 읽어준다.

**Why this priority**: 웹 접근성은 중요하지만 시각적/기능적 개선이 선행되어야 접근성 적용이 의미를 갖는다.

**Independent Test**: 키보드만으로 모달을 열고, 내부 요소를 탐색하고, 닫을 수 있으면 성공.

**Acceptance Scenarios**:

1. **Given** 모달이 열린 상태, **When** Tab 키를 반복 입력, **Then** 포커스가 모달 내 요소만 순환하며 배경으로 빠져나가지 않는다
2. **Given** 모달이 열린 상태, **When** 스크린 리더가 활성화됨, **Then** 모달 역할(dialog)과 제목이 읽힌다

---

### Edge Cases

- WASD 이동 중 화면 밖(탭 전환 등)으로 포커스가 빠질 때 → 키 해제 이벤트가 누락되어 이동이 멈추지 않는 경우 처리
- 매우 먼 거리까지 이동했을 때 → 렌더링 정밀도(floating point) 문제가 발생하지 않도록 이동 범위 상한 설정
- 모달 내에서 3D 캔버스(행성 프리뷰)가 포함된 경우 → 포커스 트랩과 3D 인터랙션 충돌 방지
- 장식 요소가 은하(행성)와 겹쳐 클릭을 방해하는 경우 → 장식 요소는 포인터 이벤트를 투과시켜야 함
- 저사양 기기에서 장식 요소가 성능을 저하시키는 경우 → 장식 요소 수를 줄이거나 LOD(Level of Detail) 적용

## Requirements *(mandatory)*

### Functional Requirements

**WASD 자유이동**

- **FR-001**: 카메라 전환(은하 클릭)이 완료된 후에는 시스템이 카메라 위치를 자동으로 변경하지 않아야 한다
- **FR-002**: WASD 이동으로 도달한 위치는 키를 놓은 후에도 유지되어야 한다
- **FR-003**: 은하 클릭 시 카메라가 현재 위치에서 목표 위치로 부드럽게 전환되어야 한다
- **FR-004**: 카메라 전환 애니메이션 중에는 WASD 입력이 무시되어야 한다
- **FR-005**: 카메라 이동 범위는 우주 공간의 합리적인 경계 내로 제한되어야 한다
- **FR-005a**: 은하 상세 뷰에서 뒤로 나올 때 카메라는 은하를 클릭하기 직전의 WASD 위치로 복귀해야 한다

**모달 리디자인**

- **FR-006**: 모든 모달은 shadcn Dialog 컴포넌트 기반으로 통일되어야 한다
- **FR-007**: 모달 크기는 콘텐츠 유형에 따라 차등 적용되어야 한다 (소형: 로그인/은하 생성, 대형: 게시글 작성/상세)
- **FR-007a**: 게시글 작성 모달은 좌우 2단 레이아웃이어야 한다 (좌: 3D 프리뷰+커스터마이즈 패널, 우: 제목+본문 입력)
- **FR-008**: 모든 모달에 시각적 닫기 버튼(X)이 제공되어야 한다
- **FR-009**: 모달은 열림/닫힘 시 부드러운 전환 애니메이션을 가져야 한다
- **FR-010**: 모달은 768px 미만 화면에서 폭이 자동 조정되어야 한다
- **FR-011**: 모달이 열려 있는 동안 배경 스크롤이 방지되어야 한다
- **FR-012**: 모달 내 포커스 트랩이 동작하여 키보드 접근성을 보장해야 한다

**우주 장식 강화**

- **FR-013**: 성운(네뷸라) 효과가 우주 공간 곳곳에 배치되어야 한다 — 은은하고 반투명한 파스텔톤 파티클 구름 스타일
- **FR-014**: 소행성대가 현재보다 넓은 범위에 분산 배치되어야 한다
- **FR-015**: 원거리 배경 은하(장식용)가 추가되어 깊이감을 제공해야 한다
- **FR-016**: 모든 장식 요소는 사용자의 은하(행성) 클릭을 방해하지 않아야 한다
- **FR-017**: 장식 요소 추가 후에도 60fps 렌더링 성능이 유지되어야 한다

### Key Entities

- **카메라 상태**: 현재 위치, 목표 위치, 전환 중 여부 — WASD 이동과 은하 전환의 기준
- **모달 변형**: 모달 유형(소형/대형)과 각 유형별 크기·레이아웃 설정
- **장식 요소**: 성운, 소행성, 배경 은하 등 — 위치, 크기, 애니메이션 속성

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: WASD로 이동한 후 키를 놓고 10초 경과 시 카메라 위치 변화가 0이다
- **SC-002**: 게시글 작성 모달에서 3D 프리뷰와 커스터마이즈 패널이 스크롤 없이 한 화면에 보인다 (1280px 이상 화면 기준)
- **SC-003**: 모든 모달에서 열림/닫힘 애니메이션이 동작하고, X 버튼·ESC·배경 클릭 3가지 닫기 방법이 모두 작동한다
- **SC-004**: 우주 뷰에서 카메라를 임의 방향으로 회전했을 때, 모든 사분면에 최소 1개 이상의 시각 요소(성운/소행성/별)가 보인다
- **SC-005**: 장식 요소 추가 후 평균 프레임 레이트가 55fps 이상 유지된다
- **SC-006**: Tab 키로 모달 내부 요소만 순환할 수 있고, 배경 요소로 포커스가 빠지지 않는다

## Clarifications

### Session 2026-03-27

- Q: 은하 상세 뷰에서 뒤로 나올 때 카메라 위치는? → A: WASD로 이동하던 직전 위치로 복귀
- Q: 게시글 작성 모달의 레이아웃 구성은? → A: 좌우 2단 (좌: 3D 프리뷰+커스터마이즈, 우: 제목+본문 입력)
- Q: 성운(네뷸라) 시각 스타일은? → A: 은은하고 반투명한 파스텔톤 파티클 구름

## Assumptions

- 기존 shadcn/ui 설치가 완료되어 있으며 Dialog 컴포넌트를 추가 설치하여 사용 가능하다
- WASD 이동 범위 상한은 우주 공간 반경 200 단위로 설정한다 (현재 별 필드 반경 100의 2배)
- 장식 요소는 기하학적 3D 메시(sphere, torus, particles)로 구현하며 외부 이미지 에셋 없이 코드로 생성한다
- 성능 테스트 기준은 데스크톱 Chrome 브라우저(중간 사양 GPU)이다
- 기존 테마(다크 우주 테마)와 디자인 토큰을 유지한다
