## User Scenarios & Testing *(mandatory)*

### User Story 1 - 은하계(주제) 탐색 (Priority: P1)

사용자가 앱에 접속하면 3D 우주 공간에 여러 은하계가 펼쳐져 있다. 각 은하계는 하나의 게시판 주제를 나타낸다. 사용자는 마우스/터치로 우주 공간을 자유롭게 탐색하며, 원하는 은하계를 클릭하여 해당 주제의 게시글 목록(행성들)을 볼 수 있다.

**Why this priority**: 3D 우주 탐색이 이 프로젝트의 핵심 경험이며, 은하계 없이는 게시글을 분류하고 탐색할 수 없다.

**Independent Test**: 앱 접속 후 3D 공간에서 은하계들을 확인하고, 하나를 클릭하여 진입할 수 있으면 테스트 통과.

**Acceptance Scenarios**:

1. **Given** 사용자가 앱에 처음 접속했을 때, **When** 화면이 로딩되면, **Then** 3D 우주 공간에 기존 은하계(주제)들이 시각적으로 표시된다
2. **Given** 은하계들이 표시된 상태에서, **When** 사용자가 특정 은하계를 클릭하면, **Then** 카메라가 해당 은하계로 줌인되며 내부의 행성(게시글)들이 보인다
3. **Given** 은하계 내부에 있을 때, **When** 사용자가 뒤로가기/줌아웃 동작을 하면, **Then** 전체 우주 공간 뷰로 복귀한다

---

### User Story 2 - 행성(게시글) 작성 및 조회 (Priority: P1)

사용자는 은하계 내부에서 새로운 행성(게시글)을 생성할 수 있다. 행성을 클릭하면 게시글 내용을 읽을 수 있다. 각 행성은 고유한 외형을 가지며, 은하계 내에서 배치된다.

**Why this priority**: 게시글 작성/조회는 게시판의 가장 기본적인 기능이다.

**Independent Test**: 은하계에 진입 후 게시글을 작성하고, 생성된 행성을 클릭하여 내용을 확인할 수 있으면 테스트 통과.

**Acceptance Scenarios**:

1. **Given** 사용자가 은하계 내부에 있을 때, **When** 게시글 작성 버튼을 누르고 제목/내용을 입력 후 제출하면, **Then** 새로운 행성이 은하계 내에 생성되어 나타난다
2. **Given** 은하계 내에 행성들이 있을 때, **When** 사용자가 특정 행성을 클릭하면, **Then** 해당 게시글의 제목과 내용이 표시된다
3. **Given** 게시글 조회 화면에서, **When** 사용자가 닫기 동작을 하면, **Then** 은하계 내부 뷰로 돌아간다

---

### User Story 3 - 별(좋아요) 부여 및 시각적 표현 (Priority: P1)

사용자는 마음에 드는 게시글(행성)에 별을 줄 수 있다. 별을 받은 행성 주위에는 받은 별 개수만큼 작은 별 오브젝트가 생성되어 행성 주변을 회전한다. 별이 많은 행성일수록 더 화려하게 보인다.

**Why this priority**: 별 시스템은 이 프로젝트만의 독창적 핵심 기능이며, 사용자 상호작용과 시각적 피드백의 중심이다.

**Independent Test**: 게시글에 별을 부여한 뒤, 행성 주위에 별 오브젝트가 회전하며 표시되는지 확인하면 테스트 통과.

**Acceptance Scenarios**:

1. **Given** 사용자가 게시글(행성)을 조회 중일 때, **When** 별 주기 버튼을 누르면, **Then** 해당 행성의 별 수가 1 증가하고, 행성 주위에 회전하는 별 오브젝트가 하나 추가된다
2. **Given** 별이 5개인 행성이 있을 때, **When** 사용자가 은하계 내부에서 해당 행성을 바라보면, **Then** 5개의 별 오브젝트가 행성 주위를 회전하고 있다
3. **Given** 별이 0개인 행성과 20개인 행성이 있을 때, **When** 사용자가 두 행성을 비교하면, **Then** 별이 많은 행성이 시각적으로 더 화려하게 보인다

---

### User Story 4 - 은하계(주제) 생성 (Priority: P2)

사용자는 새로운 은하계(주제/게시판)를 생성할 수 있다. 은하계 이름과 설명을 입력하면 우주 공간에 새 은하계가 나타난다.

**Why this priority**: 주제 생성은 콘텐츠 확장에 필요하지만, 초기에는 미리 만들어진 은하계로도 서비스가 가능하다.

**Independent Test**: 은하계 생성 후 우주 공간에 새 은하계가 표시되고 진입 가능하면 테스트 통과.

**Acceptance Scenarios**:

1. **Given** 사용자가 우주 공간 뷰에 있을 때, **When** 은하계 만들기를 선택하고 이름/설명을 입력하면, **Then** 새로운 은하계가 우주 공간에 생성된다
2. **Given** 새 은하계가 생성되었을 때, **When** 해당 은하계를 클릭하면, **Then** 빈 은하계 내부로 진입할 수 있다

---

### Edge Cases

- 별이 100개 상한에 도달한 행성에 추가 별 부여 시도 시 사용자에게 안내 메시지를 표시한다
- 은하계 내 행성이 많을 경우 페이지 단위로 나누어 표시하며, 페이지 전환 시 3D 공간 내에서 부드럽게 이동한다
- 브라우저가 WebGL을 지원하지 않는 경우 어떻게 대응하는가?
- 사용자가 동시에 같은 게시글에 별을 부여할 경우 데이터 정합성은 어떻게 보장하는가?
- 3D 카메라 이동 중 클릭 이벤트가 의도치 않게 발생하는 것을 어떻게 방지하는가?

## Clarifications

### Session 2026-03-25

- Q: 같은 사용자가 같은 게시글에 별을 여러 번 줄 수 있는가? → A: 무제한 (같은 게시글에 여러 번 별 부여 가능)
- Q: 행성당 별 상한선과 다수 게시글 처리 방식은? → A: 별은 게시글당 최대 100개 제한, 게시글(행성)은 페이지네이션 방식으로 은하계 내에서 다음 페이지로 이동 가능
- Q: 게시글 내용 형식은? → A: 마크다운 (제목, 볼드, 링크 등 기본 서식 지원)
- Q: 게시글 조회 UI 형태는? → A: 오버레이 패널 (3D 배경 유지, 위에 반투명 패널로 게시글 표시)
- Q: 페이지당 행성(게시글) 수는? → A: 50개

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 3D 우주 공간을 렌더링하여 사용자가 마우스/터치로 카메라를 회전, 줌, 패닝할 수 있어야 한다
- **FR-002**: 시스템은 은하계(주제)를 3D 오브젝트로 표현하고, 이름 라벨을 함께 표시해야 한다
- **FR-003**: 시스템은 은하계 클릭 시 카메라를 해당 은하계 내부로 부드럽게 전환해야 한다
- **FR-004**: 시스템은 은하계 내부에서 행성(게시글)들을 3D 오브젝트로 배치하여 표시해야 한다
- **FR-005**: 사용자는 게시글(행성)을 생성할 수 있어야 하며, 제목과 마크다운 형식의 내용을 입력할 수 있어야 한다
- **FR-006**: 사용자는 행성을 클릭하여 게시글 제목과 내용을 3D 배경 위 오버레이 패널(반투명)로 조회할 수 있어야 한다
- **FR-007**: 사용자는 게시글에 별(좋아요)을 부여할 수 있어야 하며, 같은 게시글에 횟수 제한 없이 여러 번 부여할 수 있어야 한다. 단, 게시글당 별은 최대 100개까지 부여 가능하다
- **FR-008**: 행성 주위에 해당 게시글이 받은 별 개수만큼 별 오브젝트가 생성되어 회전해야 한다 (최대 100개)
- **FR-013**: 은하계 내부에서 행성(게시글)은 페이지당 50개씩 표시하며, 페이지네이션 방식으로 다음/이전 페이지로 이동하여 추가 행성들을 탐색할 수 있어야 한다
- **FR-009**: 사용자는 새로운 은하계(주제)를 생성할 수 있어야 하며, 이름과 설명을 입력할 수 있어야 한다
- **FR-010**: 시스템은 은하계 내부에서 우주 공간 뷰로 복귀하는 네비게이션을 제공해야 한다
- **FR-011**: 시스템은 게시글/은하계 데이터를 영구적으로 저장하고 불러올 수 있어야 한다
- **FR-012**: 별이 많은 행성은 시각적으로 더 화려하게 표현되어야 한다 (발광 효과, 별 밀도 등)

### Key Entities

- **은하계(Galaxy)**: 게시판 주제를 나타냄. 이름, 설명, 생성일, 포함된 행성 목록을 가짐
- **행성(Planet)**: 개별 게시글을 나타냄. 제목, 내용, 작성자, 작성일, 소속 은하계, 받은 별 수를 가짐
- **별(Star)**: 게시글에 대한 좋아요를 나타냄. 부여한 사용자, 대상 행성, 부여일을 가짐

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 앱 접속 후 3초 이내에 3D 우주 공간과 은하계들을 볼 수 있다
- **SC-002**: 은하계 클릭부터 내부 행성 표시까지 1초 이내에 전환된다
- **SC-003**: 별 부여 후 1초 이내에 행성 주위에 새 별 오브젝트가 나타나 회전한다
- **SC-004**: 행성 50개, 별 총 500개가 있는 은하계에서 30fps 이상의 프레임레이트를 유지한다
- **SC-005**: 사용자가 처음 방문해도 별도 안내 없이 은하계 탐색, 게시글 조회, 별 부여를 수행할 수 있다

## Assumptions

- 사용자는 WebGL을 지원하는 모던 브라우저(Chrome, Firefox, Safari, Edge 최신 2개 버전)를 사용한다
- 데스크톱 브라우저를 우선 지원하며, 모바일 대응은 후속 단계에서 진행한다
- 초기 버전에서는 게시글 수정/삭제 기능은 범위에 포함하지 않는다
- 데이터 저장은 서버 기반 영구 저장소를 사용한다
- 사용자 인증은 간단한 닉네임 입력 방식을 기본으로 하며, 별도 로그인 시스템은 후속 단계에서 구현한다
